In this blog assignment, you will conduct simulation for two-dimensional heat diffusion in several ways.

In the lecture note, we represented steps to simulate heat diffusion as a sequence of matrix-vector multiplications. Let’s expand it to two dimensions.

In two-dimensions, the heat equation reads \[\frac{\partial f(x, t)}{\partial t} = \frac{\partial^2 f}{\partial x^2 } + \frac{\partial^2 f}{\partial y^2 }\;.\] Using the similar discretization scheme as in one-dimensional case, let’s put \[ x_i = i \Delta x,\;\; y_j = j \Delta y,\;\; t_k = k \Delta t, \] for \(i = 0, \cdots, N-1\); \(j = 0, \cdots, N-1\); and \(k = 0, 1, 2 \cdots\).

When we define \(u_{i, j}^k = f(x_i, y_j, t_k)\), the update equation in discrete time can be reprented as \[ u_{i, j}^{k+1} \approx u_{i, j}^k + \epsilon \left(u_{i+1, j}^k + u_{i-1, j}^k + u_{i, j+1}^k + u_{i, j-1}^k - 4 u_{i, j}^k\right), \] where \(\epsilon\) is a small parameter. The boundary condition can be constructed to allow heat to escape: \[ u_{-1, j}^k = u_{N, j}^k = u_{i, -1}^k = u_{i, N}^k = 0. \] Note that we are not explicitly creating space for \(u_{-1, j}^k\), \(u_{N, j}^k\), \(u_{i, -1}^k\), or \(u_{i, N}^k\).

1. With matrix multiplication

As in the linear algebra lecture, represent the second term of the update equation in a matrix-vector multiplication form so that each iteration of update is:

v = v + epsilon * (A @ v.flatten()).reshape((N, N))

That is, we view \(u_{i, j}^k\) as the element with index \(N \times i + j\) in the vector.

2. Sparse matrix

In fact, the performance of Part 1 is supposed to be excruciatingly slow. While we can use the underlying optimized matrix multiplication routine from BLAS (basic linear algebra subprograms), it is not particularly effecive here, as the matrix A has less than \(5N^2\) nonzero elements out of \(N^4\) elements. Most of operations are wasted for computing zeros. Let’s use the data structure that exploits this fact: sparse matrix data structures. The scipy package holds useful sparse matrices. Since we know that the matrix A is pentadiagonal, we can use dia_array to only use \(O(N^2)\) space for the matrix, and only take \(O(N^2)\) time for each update.
